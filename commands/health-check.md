## 前置：读取规范

1. 读 `~/Opensource/CLAUDE.md`（工作区统一规范）
2. 读 `~/Opensource/ai-dev-guide.md`（详细开发规范 + 项目模板）

## 检查项

按以下清单逐项检查，每项标记 ✅（通过）、⚠️（警告）、❌（不通过）：

### 1. 指令文件

- [ ] `CLAUDE.md` 存在
- [ ] `CLAUDE.md` 包含必要章节：概述、技术栈、项目结构、开发命令
- [ ] `AGENTS.md` 存在且是指向 `CLAUDE.md` 的**软链接**（不是独立文件或硬链接）
  - 例外：vendor 项目如果自带独立 AGENTS.md 则跳过此项

### 2. 版本管理

- [ ] `.mise.toml` 存在（Rust 纯项目例外，由 rustup 管理）
- [ ] `.mise.toml` 中锁定的版本与当前 `mise` 实际使用的版本一致

### 3. Git 规范

- [ ] 项目在 git 仓库中（`git rev-parse --git-dir` 成功）
- [ ] `.gitignore` 存在
- [ ] `.gitignore` 包含基本条目（node_modules/、.env、.DS_Store 等，根据技术栈判断）
- [ ] 没有大文件（>5MB）被 git 跟踪（`git ls-files` 检查）
- [ ] 没有敏感文件被跟踪（.env、credentials、密钥文件等）

### 4. 活跃文档（如果项目有开发计划）

- [ ] `CLAUDE.md` 中有 `## 活跃文档` 小节
- [ ] 活跃文档清单中列出的每个文件实际存在
- [ ] 如有 `STATUS.md`：最后更新日期不超过 7 天（⚠️ 警告级别，不阻塞）
- [ ] 如有 `PLAN.md`：包含阶段计划和任务拆分

### 5. 首次运行记录

- [ ] `CLAUDE.md` 中有 `## 首次运行记录` 小节（vendor/fork 项目必须有）
- [ ] 记录包含：日期、状态、踩坑点

### 6. 首次发布记录（如果已推送到 GitHub）

检查 `git remote -v` 是否有 origin：
- 如果有 remote：
  - [ ] `CLAUDE.md` 中有 GitHub 仓库地址
- 如果没有 remote：
  - 标记为 ℹ️（提示可运行 `/publish`）

### 7. 工作区索引

- [ ] 项目出现在 `~/Opensource/CLAUDE.md` 的 `## 项目索引` 表格中
- [ ] 索引中的路径与实际路径一致

## 自动修复

报告输出后，**询问用户是否要自动修复**可以自动修复的问题：

可自动修复的项目：
- 创建 AGENTS.md 软链接
- 创建缺失的 .gitignore（用合理默认值）
- 在工作区索引中添加项目
- 在 CLAUDE.md 中补充缺失的章节骨架

不能自动修复的项目（仅提示）：
- STATUS.md 过期（需要用户填写实际进度）
- 大文件被 git 跟踪（需要用户决定处理方式）
- 敏感文件被跟踪（需要用户确认）
